Angular Signals

By: Juan Munoz Escobar

The Problem

  • Angular relied on Zone.js to detect when state changed
  • Zone.js watches everything clicks, timers, HTTP calls
  • Any event triggered a check of the entire component tree
  • Works fine for small apps gets expensive at scale

What is a Signal?

A reactive value that Angular can track precisely only what depends on it updates.

  • signal() creates a reactive value Angular can watch
  • computed() derives a value from signals, auto-updates when dependencies change
  • effect() runs side-effect code whenever a signal it reads changes

signal()

Before plain variable

export class CounterComponent {
  count = 0;

  increment() {
    this.count++;
  }
}

After signal

export class CounterComponent {
  count = signal(0);

  increment() {
    this.count.update(v => v + 1);
  }
}

In templates, call the signal like a function: {{ count() }}

computed()

export class CartComponent {
  price    = signal(29.99);
  quantity = signal(3);

  total = computed(() => this.price() * this.quantity());
}
  • total is read-only meaning you cannot set it directly
  • Angular only recomputes it when a dependency actually changes
  • No manual ngOnChanges or subscription logic needed

effect()

Used for side effects that depend on signals

export class ThemeComponent {
  darkMode = signal(false);

  constructor() {
    effect(() => {
      localStorage.setItem('theme', this.darkMode() ? 'dark' : 'light');
    });
  }
}
  • Good for logging, syncing to localStorage, analytics
  • Automatically cleans up when the component is destroyed

Updating Signals set() vs update()

name = signal('John');


this.name.set('Jane');             


this.name.update(n => n + ' Doe');
  • set(value) use when you already have the new value ready
  • update(fn) use for increments, toggles, or transforms

Why It Matters

  • Only the components that changed get re-rendered, not the entire tree
  • State changes are explicit and easy to follow
  • Less reliance on RxJS for simple state management

Summary

Using Signals, Angular knows exactly what changed and only updates that.
